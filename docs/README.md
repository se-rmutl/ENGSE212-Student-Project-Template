# โครงสร้างเอกสาร V&V

แต่ละโฟลเดอร์เก็บหลักฐานตามลำดับการทำงาน เพื่อทำให้ตรวจสอบย้อนกลับได้และง่ายต่อการประเมิน

| Folder | จุดประสงค์ |
|---|---|
| `00-project-context` | บริบทโครงงาน ทีม และเป้าหมายคุณภาพ |
| `01-requirements` | requirement baseline, acceptance criteria, requirement review |
| `02-traceability` | requirement traceability matrix |
| `03-test-design` | test plan, test cases, system/UAT/API scenarios |
| `04-test-data` | test data และการตรวจสอบข้อมูล |
| `05-non-functional` | NFR outline และผลการทดสอบเชิงไม่เป็นฟังก์ชัน |
| `06-defects` | defect log, triage, retest evidence |
| `07-test-reports` | test metrics, test summary report |
| `08-ai-use` | AI use declaration และ AI audit trail |
| `09-reflection` | peer review, individual reflection, learning log |
